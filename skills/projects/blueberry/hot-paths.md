# Blueberry — Hot Paths

Code paths where extraction, indirection, or allocation per call has historically caused frame-time regressions. **Default: decline CodeRabbit extraction suggestions for code in these paths**; reply with rationale citing this skill.

This list grows as we observe regressions. Generated by `record-learning` from observed PR outcomes.

<terrain_generation>
- `crates/blueberry-terrain/src/canyon.rs` — canyon generator. Per-chunk hot loop.
- `crates/blueberry-terrain/src/chunks.rs` — chunk generation pipeline.
- `crates/blueberry-terrain/src/heightmap.rs` — heightmap sampling.
- `crates/blueberry_terrain_foundation/` — foundation crate (low-level math, called per-vertex).

Pattern: any function inside the chunk-generation traversal. If you're inside `for chunk in ...` or its descendants, you're hot.
</terrain_generation>

<rendering_per_frame>
- `src/moebius/` — Moebius post-process. Called once per frame, sometimes more for multiple passes.
- `src/sdf/grid.rs` — SDF grid traversal for visible props (the meshing path is one-shot, but evaluation during traversal can be hot if many props).

Pattern: anything inside `Update` or `PostUpdate` system schedules running every frame on a per-entity basis.
</rendering_per_frame>

<physics>
- `src/agent_tools/` physics-raycast methods — called per BRP request, but BRP requests can fire from automation loops.
- Rapier collision callback paths (when used for gameplay).

Pattern: any `fn(...) -> bool` that's invoked from inside Rapier's broadphase or narrowphase.
</physics>

<not_hot>
Cold paths where extraction is fine:
- `src/main.rs` — startup wiring.
- Plugin builders (`fn build(&self, app: &mut App)`).
- Asset loading code.
- Inventory authoring (`src/inventory_atelier.rs`).
- Editor / debug code in `src/agent_tools/`.
- Test code.
- Anything under `journal/nightly/` (Python pipeline; not runtime).
- `ops/` Docker scheduled job code.
</not_hot>

<diagnostic_workflow>
When unsure if a path is hot, before declining a CodeRabbit suggestion:
1. Check this list. If listed, decline.
2. If not listed but the function is inside an `Update`/`PostUpdate` system schedule, treat as hot.
3. If still unsure, the Manager (or a benchmark Picker) can run `cargo bench` before and after. Path becomes hot if extraction shows >1% regression.
</diagnostic_workflow>
